Schedule: midnight UTC daily. Can also be triggered manually.
Action

Quality Review

1. What breaks?

SO workflow fails on form submission (Step 1): No record, no DM, no ClickUp task. Signal: absence of #ops-digest notification. Student can re-submit (duplicate check handles gracefully). Check n8n execution logs.
SA workflow fails on approval (Step 3): Student stuck in "approved" state with no Creator ID. No welcome DM sent. Check n8n logs, re-trigger SA manually if needed.
SO runs twice on same submission (Step 1): Duplicate check catches it — sends "already applied" DM. No double records in Supabase.
SA fires on duplicate webhook event (Step 3): State machine guard catches it — only pending→approved transition proceeds. Already-approved students are no-ops.
BE workflow fails on enrollment (Step 4): No enrollment created, no DM sent. Student can re-submit (duplicate check handles gracefully). Check n8n execution logs.
DD workflow fails on dashboard delivery (Step 5): Student doesn't get dashboard link. Zak can re-trigger by toggling ClickUp task status back and forward, or send DM manually.
Whop API unreachable during daily check (Step 6): No status changes made, no data corruption. last_checked timestamps go stale — visible in Ops Digest.

2. Who knows?

SO/SA workflow failures: #ops-digest silence is the signal. Every successful completion posts a notification. Absence = check n8n logs.
BE/DD workflow failures: Same pattern — #ops-digest posts on success. Missing notification after enrollment or dashboard delivery = check n8n logs.
Whop check failures: Ops Digest shows check freshness. Stale last_checked timestamps = Zak investigates.
Student rejection: Only Karlo knows (manual DM). No system record of rejection reason — just ClickUp task → Rejected status.
Wise setup request: Karlo tracks in Discord DMs. Task stays pending until student confirms Wise account.

3. What's the fallback?

SO workflow down: Zak uses bypass path (manual trigger) to onboard student directly with pre-filled data.
SA workflow down: Zak manually generates Creator ID, updates Supabase, and sends welcome DM.
BE workflow down: Student can re-submit enrollment form. Duplicate protection prevents double-enrollments. Zak can manually create enrollment in Supabase and send DM.
DD workflow down: Zak sends dashboard link to student manually via Discord DM. Updates enrollments.dashboard_url in Supabase directly.
Whop API down: Students aren't paused/restored until next successful run. No data corruption. Monitored via Ops Digest.
Discord bot unreachable: Karlo can send DMs manually. Bot failure doesn't block the approval flow — just delays notification.
