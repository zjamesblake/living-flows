Quality Review

1. What breaks?

1.0v2 fails on submission (Step 1): No ClickUp task, no confirmation DM. Signal: #ops-digest silence. Student can re-submit — duplicate handling is safe.
2.0v2 or 3.0v2 Gemini call fails (Steps 2–3): Submission stuck in "technical qa" or "brand qa" status. No bad data written. Check n8n logs, re-trigger the workflow.
4.2 revision DM fails (Step 4a): Student doesn't know what to fix. Task shows "Revision Requested" but no DM received. Karlo can manually relay feedback from the ClickUp task.
5.0v2 grading fails (Step 5): Submission approved but ungraded. Media buyer not notified. Re-trigger 5.0v2.
Duplicate submission (same files): Each gets its own QA run. Not deduplicated — acceptable at current volume.

2. Who knows?

All workflow completions post to #ops-digest. Silence = failure. Check n8n execution logs.
QA results visible on ClickUp task + Ops Dashboard. Per-gate pass rates tracked per brand.
Karlo sees all submissions in ClickUp and can spot stuck items (tasks sitting in a status too long).
Override rate tracked on Ops Dashboard. High override rate for a brand = prompts need adjustment.

3. What's the fallback?

Any workflow down: Karlo can manually approve via 4.1 as an interim measure. QA results still in ClickUp + Supabase once workflows resume.
Gemini API down: All QA pauses. No bad data written. Resume when API returns — submissions stay in their current status.
Discord bot down: DMs delayed but QA pipeline keeps running. All results still recorded in ClickUp + Supabase. Karlo can relay feedback manually.
QA gates miscalibrated for a brand: Zak adjusts prompts in Supabase. Re-run affected submissions. Max 2 adjustment rounds — if still off, re-do brand interview.
