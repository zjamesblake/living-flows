Test submission method: Use a dedicated CREATOR-TEST student record. Keeps the QA pipeline unchanged — no bypass needed.
Calibration Loop: After initial QA testing, brand reviews results. If inconsistencies found (same creative passes then fails = interpretation issue), we enter the prompt calibration cycle:
1. Brand provides feedback on borderline cases (which should pass/fail and why)
2. Zak categorizes feedback by gate (B1-B4) and determines which prompt clause to adjust
3. Targeted prompt edits made in Supabase (never full replacement — surgical fixes)
4. Re-run the specific borderline creatives through the test script (3-5 runs per creative for consistency check)
5. If consistency is 80%+ and brand is satisfied → proceed to Step 20

Max 2 calibration rounds. If prompts still inconsistent after 2 rounds → re-do the interview for deeper data. Each creative is run 5-10 times to gauge consistency — different results on the same input means an interpretation issue exists.

Quality Review

1. What breaks?

Brand never responds to outreach (Steps 4→5): 48hr → 5 day → final ping follow-up cadence. Then → Delayed.
Brand never completes interview (Step 11): 3 day → 7 day → 14 day escalation. Highest friction point in the flow.
Brand never signs contract (Steps 6→7): Same follow-up cadence. Ghost → Delayed.
Test ads reveal bad QA gates (Step 19): Max 2 adjustment rounds. If still wrong → re-do interview for deeper data.
Gemini fails during Workflow C (Step 12): Re-submit the interview JSON (idempotent). P1 fix: add failure notification to #ops-digest.
Workflow B runs twice (Step 10): Brand record is safe (UNIQUE brand_id). But Drive folder and ClickUp list would duplicate. Manual cleanup needed — rare scenario.

2. Who knows?

Workflow A or B fails silently: Brand submits form but nothing happens. All workflows now post to #ops-digest on success — absence of notification = something failed. Check n8n execution logs.
Brand stalls between Steps 4-7: No automation — relies on Karlo's follow-up cadence in the group chat. Group chat makes it visible to the whole team.
Angie needs to send PandaDoc: Triggered by Karlo messaging her. No system notification. Acceptable at current volume.
Brand status drift (Supabase vs ClickUp): Both must be updated at go-live (Step 22). Manual sync — source of truth is Supabase.

3. What's the fallback?

Interview prompt .txt creation fails: Karlo can paste the prompt template manually into a file. Template exists in master_interview_prompt.md.
Any workflow fails: All form links and backup links are in ClickUp task comments. Karlo can share manually.
Gemini prompt generation fails: Re-submit interview JSON. Upserts are idempotent — no duplicates.
PandaDoc fails: Send PDF contract manually via group chat.
Discord bot unreachable: All communication is in the group chat (human-driven). Bot failure doesn't block the flow.
